用例格式指引

字段必填说明示例
接口名称✅接口的描述名称，仅做展示高德逆地码接口
描述❌补充说明根据经纬度获取地址信息
方法✅请求方法：GET / POST / PUT / DELETE / PATCHPOST
DEV URL❌开发环境地址，env=dev 时使用https://dev-api.example.com/login
UAT URL❌测试环境地址，env=uat 时使用https://uat-api.example.com/login
PRO URL❌生产环境地址，env=pro 时使用https://api.example.com/login
服务编码❌按服务过滤用例；特殊值 auto 表示被自动加载执行auto
状态❌禁用后该接口下所有用例都不会执行启用
重要：如果当前环境对应的 URL 为空，该接口下的用例会被跳过。例如 env=dev 但 url_dev 为空，则不会执行。

三、请求数据（case_data）格式

请求数据必须是合法的 JSON 格式，框架根据请求方法自动处理。
请求方法case_data 格式框架处理方式
GET{"wd": "Python"}作为 URL 查询参数发送
GET"wd=Python&page=1"自动解析为 dict 后作为参数发送
POST/PUT/DELETE/PATCH{"username":"admin","password":"123"}作为请求体（JSON body）发送
无参数{} 或留空不发送数据
自定义参数替换（#关键字#）
支持 #关键字# 占位符，运行时自动替换为动态值，减少对用例的修改。
占位符替换结果示例
#now#当前时间2026-06-06 14:30:00
#faker_name#随机中文姓名张三
#faker_ssn#随机身份证号310...
#faker_job#随机职业软件工程师
示例： 
{"username": "#faker_name#", "timestamp": "#now#"}

四、断言规则（check_rules）格式

断言规则必须是 JSON 数组，每个元素是一个断言条件。与 expected_results 必须数量相同、位置对齐。
① json — JSONPath 断言

从响应体中通过 JSONPath 提取值，与期望结果对比。

[{"json": "$.status"}, {"json": "$.data.name"}]
② exist — 字段存在断言

检查响应体中是否包含指定的 key。

[{"exist": "data"}, {"exist": "message"}]
③ not_exist — 字段不存在断言

检查响应体中不包含指定的 key。

[{"not_exist": "error"}, {"not_exist": "exception"}]
④ sql — 数据库断言

执行 SQL 查询，将查询结果字段值与期望结果逐一对比。

[{"sql": ["dev", "SELECT id, name FROM user WHERE id = 1"]}]
第一个元素是数据库环境名（对应配置中的键名），第二个是 SQL 语句。查询结果的每个字段与 expected_results 中的值逐一对比。
⑤ status_code — HTTP 状态码断言

校验 HTTP 响应状态码，值填空字符串即可。

[{"status_code": ""}]

五、期望结果（expected_results）格式

期望结果必须是 JSON 数组，与 check_rules 一一对应（数量必须相同，位置必须对齐）。
check_rulesexpected_results含义
{"json": "$.status"}"1"响应中 status 字段值应等于 "1"
{"exist": "data"}"True"响应中应包含 data 字段
{"not_exist": "error"}"True"响应中不应包含 error 字段
{"status_code": ""}200HTTP 状态码应为 200
{"sql": ["dev", "SELECT id, name FROM user WHERE id=1"]}[1, "张三"]SQL 查询结果 id=1, name="张三"

六、完整示例

接口名称：高德逆地码接口
方法：GET
DEV URL：https://ditu.amap.com/service/regeo
UAT URL：https://ditu.amap.com/service/regeo
PRO URL：https://ditu.amap.com/service/regeo

用例1：
  标题：逆地码查询成功-dev
  请求数据：{"longitude": "121.049255", "latitude": "31.315590"}
  断言规则：[{"json": "$.status"}, {"exist": "data"}]
  期望结果：["1", "True"]
  环境：dev

用例2：
  标题：带动态参数-dev
  请求数据：{"name": "#faker_name#", "time": "#now#"}
  断言规则：[{"json": "$.status"}]
  期望结果：["1"]
  环境：dev

用例3：
  标题：数据库校验-dev
  请求数据：{"order_id": "106828548100480"}
  断言规则：[{"json": "$.status"}, {"sql": ["dev", "SELECT id, BIZ_TYPE FROM ota_order WHERE id = 106828548100480"]}]
  期望结果：["1", [106828548100480, 10]]
  环境：dev

七、常见错误

类型错误写法正确说明
🔴{"json": "$.status"}✅断言规则必须是数组，即使只有一条：[{"json":"$.status"}]
🔴[{"json":"$.status"}]
["1", "True"]		❌断言1条但期望2个，数量不匹配
🔴{username: "admin"}✅JSON 的 key 必须用双引号：{"username":"admin"}
🔴{'username':'admin'}✅不能用单引号，JSON 只支持双引号
🟡接口 URL 为空⚠️对应环境的 URL 为空时，用例会被跳过不执行
🟡env 不匹配⚠️运行 env=dev 但用例 env=uat，该用例不会被执行
