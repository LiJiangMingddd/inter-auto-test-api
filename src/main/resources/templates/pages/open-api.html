接口概述

本接口用于 Agent 或其他外部系统 批量将接口定义及测试用例导入到接口中台。采用 两条腿 的认证方式：

  • Step 1 — 使用 appId + appKey 换取 Sa-Token
  • Step 2 — 在请求头携带 Authorization: token 调用批量导入接口
  • 支持事务性提交：单个接口失败不影响其他接口
  • 返回详细的成功/失败统计

认证方式（两步获取）

1 获取 AppID 和 AppKey

在「系统管理 → 用户管理」中新增用户，系统 自动生成 无规则随机字符串作为 AppID 和 AppKey。AppKey 仅在创建时展示一次，请立即复制保存！

2 调用 /api/auth/getToken 获取 Token

传入 appId + appKey，获取 Sa-Token。后续所有 API 调用在 Authorization 请求头中携带此 token。

3 调用 /api/open/batch-import 批量导入

在请求头 Authorization: {token} 中携带 token，请求体中传入接口和用例数据。

Step 1：获取 Token 接口

POST /api/auth/getToken

Content-Type: application/json; charset=utf-8 （无需 token，开放接口）

请求体：

{
  "appId":  "APP_3E8F2C1A...",  // 必填 - 用户管理中自动生成的 AppID
  "appKey": "6F8E..."            // 必填 - 用户管理中自动生成的 AppKey（明文）
}

响应示例：

{
  "code": 200,
  "msg":  "success",
  "data": {
    "token":  "f26a3e2c-8b1d-4f5e-9c7a-3d8e2f1a6b4c",  // Sa-Token 访问令牌
    "expire": 86400                                     // 有效期（秒），默认 24 小时
  }
}

Curl 示例：

curl -X POST http://localhost:8849/api/auth/getToken \
  -H "Content-Type: application/json" \
  -d '{"appId": "APP_3E8F2C1A...", "appKey": "6F8E..."}'
注意：token 有效期为 24 小时，过期后需重新调用本接口获取。如果用户被禁用，token 将立即失效。

Step 2：批量导入接口

POST /api/open/batch-import

Content-Type: application/json; charset=utf-8 请求头：Authorization: {token}

请求体结构：

{
  "interfaces": [   // 必填 - 接口列表（最少 1 条）
    {
      "apiName":      "接口名称",            // 必填 - 接口名称
      "apiInfo":      "接口描述",            // 可选 - 功能描述
      "method":       "POST",               // 必填 - GET/POST/PUT/DELETE/PATCH
      "urlDev":       "http://dev...",       // 可选 - 开发环境地址
      "urlUat":       "http://uat...",       // 可选 - 测试环境地址
      "urlPro":       "http://pro...",       // 可选 - 生产环境地址
      "serviceCode":  "user-service",       // 可选 - 所属服务编码
      "enabled":      1,                     // 可选 - 默认 1（启用），0 禁用
      "testcases": [                         // 可选 - 关联测试用例
        {
          "caseTitle":        "用例标题",             // 必填
          "caseData":         "{\"userId\": 1}",     // 可选 - 请求数据
          "checkRules":       "$.code == 200",       // 可选 - 断言规则
          "expectedResults":  "{\"code\":200}",      // 可选 - 期望结果
          "env":              "uat",                  // 可选 - dev/uat/pro
          "enabled":          1,                      // 可选
          "sortOrder":        1                       // 可选 - 排序序号
        }
      ]
    }
  ]
}

响应示例：

{
  "code": 200,
  "msg":  "success",
  "data": {
    "success":         true,    // 是否全部成功
    "interfaceSuccess": 2,      // 成功接口数
    "interfaceFailed":  0,      // 失败接口数
    "testcaseSuccess":  3,      // 成功用例数
    "testcaseFailed":   0,      // 失败用例数
    "errors": []                // 失败详情（空数组=全部成功）
  }
}
提示：如果部分接口导入失败，errors 数组会包含每条失败的具体原因。事务性保证：成功的不回滚，失败的不提交。

完整 Curl 调用流程

以下示例展示完整的 两步流程：

第一步：获取 Token

TOKEN=$(curl -s -X POST http://localhost:8849/api/auth/getToken \
  -H "Content-Type: application/json" \
  -d '{"appId": "你的AppID", "appKey": "你的AppKey"}' | jq -r '.data.token')

第二步：批量导入接口

curl -X POST http://localhost:8849/api/open/batch-import \
  -H "Content-Type: application/json" \
  -H "Authorization: $TOKEN" \
  -d '{
  "interfaces": [
    {
      "apiName": "获取用户信息",
      "method": "GET",
      "urlDev": "http://dev-api.example.com/user/info",
      "urlUat": "http://uat-api.example.com/user/info",
      "urlPro": "http://api.example.com/user/info",
      "serviceCode": "user-service",
      "testcases": [
        {
          "caseTitle": "正常请求-返回200",
          "caseData": "{\"userId\": 1}",
          "checkRules": "$.code == 200",
          "env": "uat"
        },
        {
          "caseTitle": "异常请求-用户不存在",
          "caseData": "{\"userId\": 99999}",
          "checkRules": "$.code == 404",
          "env": "uat"
        }
      ]
    },
    {
      "apiName": "创建订单",
      "method": "POST",
      "urlDev": "http://dev-api.example.com/order/create",
      "urlUat": "http://uat-api.example.com/order/create",
      "serviceCode": "order-service",
      "testcases": [
        {
          "caseTitle": "正常创建订单",
          "caseData": "{\"productId\": 1, \"quantity\": 2}",
          "env": "uat"
        }
      ]
    }
  ]
}'

错误码说明

HTTP 状态码 业务码 说明
200 200 请求成功
401 401 未授权 / appId 不存在 / appKey 校验失败 / token 过期
403 403 权限不足（无 user:manage 或 api:manage 权限）
400 — 参数校验失败（JSR303），如缺少必填字段
